## Spacing Scale

Tkinter uses pixel integers for `padx`/`pady`/`ipadx`/`ipady`. All values are multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, Checkbutton internal padding |
| sm | 8px | Button padx/pady, label-to-widget gap |
| md | 16px | Frame internal padding (all dialog frames) |
| lg | 24px | Tab content outer padding |
| xl | 32px | Dialog window padding (root padx/pady) |
| 2xl | 48px | Not used in Phase 1 |
| 3xl | 64px | Not used in Phase 1 |

Exceptions:
- Listbox height: 8 rows (fixed row count, not pixel-based)
- Toplevel minimum window width: 480px
- Toplevel minimum window height: 380px
- Touch/click target floor: all Buttons have `pady=8` minimum (ensures ~32px click height on 96 DPI)

## Typography

Tkinter font declarations use `("Segoe UI", size, weight)` tuples. Fall back to `TkDefaultFont` if Segoe UI is unavailable (handled via `tkinter.font.nametofont` check).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body / Label | 9pt | normal (400) | n/a (single line) | All ttk.Label, Checkbutton, Combobox, Listbox entries |
| Button | 9pt | normal (400) | n/a | All ttk.Button labels |
| Tab heading | 9pt | normal (400) | n/a | ttk.Notebook tab labels (OS-rendered) |
| Dialog title | 10pt | bold (700) | n/a | Section headers inside tabs (e.g. "Mapper til scanning") |

Note: Tkinter does not support fractional line-height ratios. Single-line widgets use OS default metrics. Multi-line widgets (e.g. a future status label) use `wraplength` instead of line-height.

## Color

This is a native Windows application. The color contract governs:
1. Programmatically generated tray icon (Pillow)
2. Any explicit color overrides on tkinter widgets (keep to minimum — prefer OS theme)

Widget backgrounds default to the Windows system theme (via `ttk.Style` with no explicit theme override). Do NOT set explicit background/foreground on ttk widgets unless specified below.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | OS default (SystemButtonFace) | Dialog background, frame backgrounds — let Windows theme control |
| Secondary (30%) | OS default (SystemWindow) | Listbox background, Entry background — let Windows theme control |
| Accent (10%) | #2563EB (blue) | Tray icon shield fill color ONLY |
| Icon foreground | #FFFFFF (white) | Tray icon "G" letter glyph ONLY |
| Destructive | OS default (SystemHighlight) | No destructive actions in Phase 1 — reserved for Phase 3 |

Accent reserved for: tray icon shield fill — no other widget uses #2563EB in Phase 1.

Theme override: use `ttk.Style().theme_use("vista")` on Windows to get native ttk rendering. Fall back to `"clam"` if `"vista"` is unavailable.

### Tab 0: Mapper

Source: CONTEXT.md D-05, D-08, D-09, REQUIREMENTS.md CONF-02

**Layout** (top to bottom, within `padx=16, pady=16` frame):

```
┌─────────────────────────────────────────┐
│ Label: "Mapper til scanning"            │ ← dialog title font (10pt bold)
├─────────────────────────────────────────┤
│ tk.Listbox                              │ ← 8 rows tall, full width
│   (single-selection, no border needed) │   selectmode=SINGLE
│                                         │   yscrollcommand → Scrollbar
│ ttk.Scrollbar (right side, vertical)    │
├─────────────────────────────────────────┤
│ [Tilføj mappe]    [Fjern valgt]         │ ← buttons, left-aligned, sm gap
└─────────────────────────────────────────┘
```

**Widget specifications:**

| Widget | Type | Text | Behavior |
|--------|------|------|----------|
| Section header | `ttk.Label` | "Mapper til scanning" | 10pt bold, top of frame |
| Folder list | `tk.Listbox` | — | `selectmode=tk.SINGLE`, `height=8`, horizontal scrollbar optional |
| Vertical scrollbar | `ttk.Scrollbar` | — | linked to Listbox `yscrollcommand` |
| Add button | `ttk.Button` | "Tilføj mappe" | Opens `filedialog.askdirectory()`; silently deduplicates if folder already in list |
| Remove button | `ttk.Button` | "Fjern valgt" | Disabled (`state="disabled"`) when no Listbox selection; enabled on `<<ListboxSelect>>`; removes selected entry |

**Empty state:** Listbox shows no entries. No placeholder text. "Fjern valgt" is disabled.

**Duplicate handling:** If `askdirectory()` returns a path already in the list, do nothing (silent deduplication — no error dialog).

## Interaction Contracts

### "Fjern valgt" enabled state

- Default state: `state="disabled"`
- Enabled when: `<<ListboxSelect>>` event fires AND selection index is valid
- Re-disabled when: Listbox selection is cleared or list becomes empty after removal

### Dialog open/close lifecycle

- Open: `dialog.deiconify()` if already exists, `tk.Toplevel(root)` if first open
- On "Gem og luk": save config → `dialog.destroy()` (not `withdraw` — let GC clean up)
- On window close (X): `dialog.destroy()` without saving (discard changes)
- Threading: all Toplevel creation and destruction happens on the tkinter worker thread only, scheduled via `root.after(0, ...)` from pystray callbacks

### Config save sequence

1. Read all widget values from Tab 0 (folder list), Tab 1 (age spinbox + checkboxes), Tab 2 (combobox selection)
2. Map combobox display string to integer minutes using lookup dict
3. Construct config dict matching default schema (CONTEXT.md D-17)
4. Call `ConfigStore.save()` — atomic write (temp file + `os.replace()`)
5. Destroy Toplevel

### Queue polling

- `root.after(100, _poll_queue)` — 100ms interval
- Non-blocking: `queue.get_nowait()` wrapped in `try/except queue.Empty`
- On `OpenConfigEvent`: call `open_config_dialog()`
- On `ShutdownEvent`: call `root.quit()`
