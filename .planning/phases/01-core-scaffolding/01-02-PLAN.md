**src/ui_thread.py** — tkinter worker thread per D-02, D-04, Pitfall 2:

Class `UIThread`:
- `__init__(self, event_queue: queue.Queue, config_store: ConfigStore)`: stores queue and config_store. Sets `self.root = None`, `self.config_dialog = None`.
- `start(self)`: creates `threading.Thread(target=self._run, daemon=True)` and starts it. Returns the thread object.
- `_run(self)`: 
  1. Creates `self.root = tk.Tk()`
  2. `self.root.withdraw()` — hidden root window (per Pitfall 2, never shown)
  3. Sets ttk theme: `ttk.Style().theme_use("vista")` wrapped in try/except, fallback `"clam"`
  4. Calls `self._poll_queue()` to start polling
  5. Calls `self.root.mainloop()`
- `_poll_queue(self)`:
  1. `try: event = self.event_queue.get_nowait()` / `except queue.Empty: pass`
  2. If event received, calls `self._handle_event(event)`
  3. `self.root.after(100, self._poll_queue)` — reschedule per D-04 (100ms interval)
- `_handle_event(self, event)`:
  - `isinstance(event, OpenConfigEvent)`: calls `self._open_config_dialog()` (stub in this plan — logs "Config dialog requested", full implementation in Plan 03)
  - `isinstance(event, ShutdownEvent)`: calls `self.root.quit()`
  - `isinstance(event, ScanNowEvent)`: logs `"ScanNowEvent received by UI thread"` (Phase 2 will add scan pipeline here)
- `_open_config_dialog(self)`: STUB for this plan — `logging.info("Config dialog requested")`. Plan 03 replaces this with actual dialog.

Import: `import tkinter as tk, tkinter.ttk as ttk, queue, threading, logging`

**src/scheduler.py** — ScanScheduler per D-18, D-19, D-20, SCHED-01, SCHED-02, SCHED-03:

Class `ScanScheduler`:
- `__init__(self, event_queue: queue.Queue, config_store: ConfigStore)`: stores queue and config_store. Creates `self._stop_event = threading.Event()`. Sets `self._thread = None`.
- `start(self)`: creates `threading.Thread(target=self._run, daemon=True)`, stores in `self._thread`, starts it.
- `_run(self)`:
  1. Reads `interval = self.config_store.get_config().get("scan_interval_minutes", 1440)`
  2. If `interval == 0`: logs `"Scheduler disabled (manual only mode)"` and enters a wait loop: `self._stop_event.wait()` then returns (per D-20)
  3. Converts interval to seconds: `interval_seconds = interval * 60`
  4. Loop: `while not self._stop_event.is_set():`
     - `self._stop_event.wait(timeout=interval_seconds)` — blocks up to interval, but wakes immediately if stop_event is set
     - If `self._stop_event.is_set()`: break
     - `self._trigger_scan()`
- `_trigger_scan(self)`: posts `ScanNowEvent()` to `self.event_queue`, logs `"Scheduled scan triggered"`
- `trigger_now(self)`: public method for manual scan trigger — posts `ScanNowEvent()` to queue, logs `"Manual scan triggered"`
- `stop(self)`: sets `self._stop_event`, joins `self._thread` with `timeout=5`

Import: `import threading, queue, logging`

**src/main.py** — Application entry point per ARCHITECTURE.md Startup Flow:

```python
import queue
import logging
import sys

from src.config_store import ConfigStore
from src.ui_thread import UIThread
from src.scheduler import ScanScheduler
from src.tray_app import TrayApp

def main():
    logging.basicConfig(
        level=logging.INFO,
        format="%(asctime)s [%(threadName)s] %(levelname)s: %(message)s",
    )
    logging.info("GDPR Scanner starting")

    event_queue = queue.Queue()
    config_store = ConfigStore()

    # Start UI thread (daemon) — tkinter mainloop
    ui_thread = UIThread(event_queue, config_store)
    ui_thread.start()

    # Start scheduler thread (daemon)
    scheduler = ScanScheduler(event_queue, config_store)
    scheduler.start()

    # Run tray app on main thread (BLOCKS — per D-01, pystray must own main thread)
    tray = TrayApp(event_queue, config_store)
    logging.info("Tray icon starting on main thread")
    tray.run()

    # After tray.run() returns (user clicked Afslut), clean up
    logging.info("Tray stopped, shutting down")
    scheduler.stop()
    sys.exit(0)

if __name__ == "__main__":
    main()
```

Key points:
1. `ConfigStore()` first — creates app dir and loads config
2. `UIThread.start()` second — daemon thread, creates hidden Tk root
3. `ScanScheduler.start()` third — daemon thread, reads config interval
4. `TrayApp.run()` last — BLOCKS main thread (this is the Win32 constraint from D-01, Pitfall 1)
5. After `tray.run()` returns, stop scheduler and exit
6. Logging format includes thread name for debugging threading issues
